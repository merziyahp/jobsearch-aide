# JobSearch Aide Website Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a two-page static website (landing page + privacy policy) for the JobSearch Aide Chrome extension, deployable to GitHub Pages.

**Architecture:** Vanilla HTML/CSS, no build step, no dependencies. Two HTML files share one CSS file. Deployed by pushing `website/` contents to a `gh-pages` branch. All brand tokens are CSS custom properties defined once in `style.css`.

**Tech Stack:** HTML5, CSS3, Formspree (form backend), GitHub Pages (hosting).

      <ul>

        <li>All data written by this extension goes to your own Google Sheet or your local machine — nowhere else.</li>

## Task 5: GitHub Pages deployment

**Prerequisites:** You need a GitHub account and the GitHub CLI (`gh`) installed, or you can do this via the GitHub website.

Note: This task initializes a git repository and pushes to GitHub. The project directory is currently not a git repo. The `website/` subdirectory is what gets served — the rest of the project stays in the same repo on the `main` branch.

- [ ] **Step 1: Fill in your placeholders**

  Before deploying, replace these two placeholders in `website/index.html` and `website/privacy.html`:

  - `REPLACE_WITH_YOUR_FORMSPREE_ENDPOINT` → your Formspree form URL (e.g. `https://formspree.io/f/xpwzgjkl`). Find it at formspree.io under your form's settings.
  - `REPLACE_WITH_YOUR_EMAIL` → your support email address (appears in both files and in both footers)

  ```bash
  # Verify no placeholders remain before deploying:
  grep -r "REPLACE_WITH" /Users/merziyahpoonawala/tech_projects/jd_tracker_chrome_extension/website/
  ```

  Expected: no output (all placeholders replaced).

- [ ] **Step 2: Initialize git repo**

  ```bash
  cd /Users/merziyahpoonawala/tech_projects/jd_tracker_chrome_extension
  git init
  git add .
  git commit -m "Initial commit — extension source and store listing assets"
  ```

- [ ] **Step 3: Create GitHub repo and push**

  Create a new repo on GitHub (github.com → New repository). Name it `jobsearch-aide` or similar. Then:

  ```bash
  git remote add origin https://github.com/YOUR_GITHUB_USERNAME/jobsearch-aide.git
  git branch -M main
  git push -u origin main
  ```

- [ ] **Step 4: Push website to gh-pages branch**

  ```bash
  cd /Users/merziyahpoonawala/tech_projects/jd_tracker_chrome_extension
  git subtree push --prefix website origin gh-pages
  ```

  This pushes only the `website/` directory contents to the `gh-pages` branch — so GitHub Pages serves the site from its root, not from a subdirectory.

- [ ] **Step 5: Enable GitHub Pages**

  In your GitHub repo: Settings → Pages → Source → Deploy from branch → Branch: `gh-pages` → Folder: `/ (root)` → Save.

  The site will be live at `https://YOUR_GITHUB_USERNAME.github.io/jobsearch-aide/` within 1-2 minutes.

- [ ] **Step 6: Verify live site**

  Open the GitHub Pages URL in a browser. Confirm:
  - Landing page loads with correct styles and brand colors
  - "Get early access" form submits successfully (enter a test email, check your Formspree dashboard for the submission)
  - Privacy Policy link navigates correctly
  - Site is readable on mobile (resize browser to under 768px wide)
